## Stage 4 completion

v0.5.0 completes Stage 4 orientation interoperability, provenance, acquisition, and source
comparison:

- explicit offline PDBTM API-v1 JSON/transformed-PDB pairs with direct current-coordinate
  applicability and no fitting;
- bounded direct PDBTM retrieval from the reviewed provider routes;
- a validated content-addressed local cache with atomic publication and revalidation;
- visibly separate **Fetch / Refresh** and **Use cached pair** actions, so using cached data never
  silently authorizes network access;
- cancellation, request-generation, selected-snapshot, and stale-result protections around the
  GUI worker lifecycle;
- schema 1.4 acquisition provenance for validated cached PDBTM data;
- cached-PDBTM PyMOL QC, slab rendering, and export after current-object applicability is
  independently established;
- a pure, offline-only OPM oriented-PDB adapter;
- explicit PDBTM–OPM comparison after both sources independently match the same immutable selected
  object;
- schema 1.5 comparison reports and four comparison-owned boundary visuals;
- input-coordinate preservation throughout all workflows.

The comparison reports conservative geometric evidence. It does not automatically fit or align
coordinates, select a source, produce a consensus orientation, rank providers, or decide which
source is biologically correct.

## Installation

After publication, download `MembraneVisualQC-0.5.0.zip` and
`MembraneVisualQC-0.5.0.zip.sha256` from the v0.5.0 GitHub prerelease. Verify the checksum, install
the ZIP through PyMOL Plugin Manager, completely restart PyMOL, and open
**Plugin > Membrane Visual QC**.

The wheel and source distribution are supplied as GitHub release assets for inspection and
development. The project is not published to PyPI.

## Security and privacy boundary

- Ordinary tests and package import make no live network request.
- Only explicit PDBTM **Fetch / Refresh** authorizes direct HTTPS retrieval.
- Provider host/routes, response sizes, payload roles, and payload pairing are bounded and
  validated; redirects, retries, proxy discovery, PAC, CONNECT, and proxy credentials are not
  supported.
- Reports retain safe provenance, sizes, digests, and stable typed errors rather than raw payloads,
  cache paths, credentials, proxy data, raw exceptions, usernames, hostnames, or local absolute
  paths.
- Official PDBTM and OPM payloads remain outside Git and all distribution archives.

## Known limitations

- OPM is offline-only; there is no live OPM retrieval.
- PDBTM proxy/PAC/CONNECT operation is unsupported.
- The cache has no automatic migration or garbage collection.
- There is no batch comparison.
- Only planar single-membrane models are supported; curved and multiple membranes are not.
- Comparison thresholds are geometric review bands, not biological truth.
- This is a GitHub prerelease for limited public testing, not a stable v1 contract.
- No PyPI publication is provided.
