# Membrane Visual QC v0.4.0

Status: GitHub prerelease candidate for limited public testing. Publication has not occurred.

## Highlights

- Offline PDBTM API-v1 orientation import using an explicit local JSON record and its matching
  transformed-PDB companion.
- Direct identity or analytical-inverse coordinate applicability without structural fitting.
- New PyMOL commands and a third GUI mode for current-frame membrane rendering and QC.
- Immutable report schema 1.3 with exact payload, provider, adapter, geometry, matrix, precision,
  threshold, fingerprint, and coordinate-match provenance.
- Deterministic lifecycle cleanup, schema-1.3 Context OFF/ON support, and unchanged CSV columns.

Install the ZIP through PyMOL Plugin Manager, fully restart PyMOL, then open
**Plugin > Membrane Visual QC**. Verify the ZIP against its checksum sidecar. The wheel and source
distribution are GitHub release assets for inspection and development; no PyPI publication is
planned for this prerelease.

## Offline PDBTM usage

Load exactly one current, single-state PyMOL molecular object. Select two matching local provider
files: the PDBTM API-v1 JSON record and its transformed-PDB companion. Applicability is checked
against the complete containing object even when QC targets a chain or residue selection inside
that object. Current coordinates must directly match either the transformed companion or its
analytically inverse-transformed reference. The plugin never fits, rotates, translates, renames,
duplicates, or deletes the input object.

The GUI mode is **PDBTM offline pair**. Equivalent commands are:

```pml
mvqc_check_pdbtm selection=my_object, pdbtm_json=C:/payloads/record.json, transformed_pdb=C:/payloads/record.trpdb
mvqc_slab_pdbtm selection=my_object, pdbtm_json=C:/payloads/record.json, transformed_pdb=C:/payloads/record.trpdb
```

`mvqc_check_pdbtm` supports Context OFF/ON and produces schema 1.3 after successful resolution.
`mvqc_slab_pdbtm` renders boundaries without creating an exportable QC report.

## Report schema 1.3

Schema 1.3 is the immutable v0.4.0 PDBTM provenance contract. JSON Schema enforces its structural
contract; every schema-1.3 report must then pass the mandatory Stage 4 semantic validator for
finite unit normals, reviewed source geometry, symmetric offsets, and resolved/current geometry
agreement. Schemas 1.0–1.3 are immutable after publication. CSV columns are unchanged.

## Scientific and operational limitations

- Geometric applicability is not biological correctness.
- Provider Side1/Side2 labels are not converted into inside/outside biology.
- Ordinary SASA is not lipid accessibility.
- Local chemical-context labels are conservative visual-review evidence, not energetic or
  mechanistic conclusions.
- Reports are visual-QC evidence, not definitive structural validation.
- There is no network retrieval, OPM adapter, cross-source comparison, curved or multiple membrane
  model, batch CLI, model-to-model comparison, automatic fitting/alignment, or automatic biological
  verdict.
- Slab planes can have relatively low contrast on dark backgrounds; this is a non-blocking
  pre-v1.0 UI backlog item.

## Upgrade notes from v0.3.0

Existing global-z, planar orientation-file, schema-1.1, schema-1.2, exposure, and local-context
workflows remain available. Offline PDBTM is opt-in and requires explicit local files; no existing
command is redirected to it. Schema dispatch remains 1.1 for context-disabled manual/planar runs,
1.2 for Stage 3 exposure/context, and 1.3 for resolved PDBTM orientation with Context OFF or ON.
